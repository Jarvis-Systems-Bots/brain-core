	•	<purpose>: фіксує єдину мету профілю, щоб усі компілятори збирали однаковий смисл для різних агентів.
	•	<iron_rules>: жорсткі заборони/вимоги з наслідками при порушенні. Це “не ламати прод”, “не вигадувати факти”, “виводити патчі”.
	•	<guidelines>: м’які стандарти — планування, мінімальність змін, мова відповіді тощо.
	•	<style>: стиль і формат — щоб вихід завжди був однаковим і читабельним, незалежно від агента.
	•	<response_contract>: контракт на форму відповіді. Це критично для сумісності та подальшого автоматизованого парсингу.
	•	<determinism>: вимога стабільності — без випадкових порядків, щоб дифи та секції не “скакали” між запусками.
